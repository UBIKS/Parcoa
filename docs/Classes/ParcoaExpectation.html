Overview

An immutable expectation object that describes what a parser expected to find.

Tasks

Properties

  •   charactersRemaining

    The number of input characters remaining.

         property
  •   expected

    A string describing what the parser expected. If the parser failed then this explains what text would have satisfied the parser, otherwise it explains what input would have allowed the parser to match more input than it did.

         property
  •   children

    An array containing ParcoaExpectation children.

         property
  • – minCharactersRemaining

    The minimum value of charactersRemaining for this context and all its chilsren. This property is memoized such that subsequent calls are costless.

Special Expectation Strings

  • + unsatisfiable

    An expectation string indicating that there exists no input that would have allowed the parsers to consume more input.
  • + choice

    An expectation string indicating that the parser had a choice between many parsers. The parser which could have consumed the most input can be found by inspecting the choice expectation’s children.
